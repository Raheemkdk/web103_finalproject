# User Stories

Reference the Writing User Stories final project guide in the course portal for more information about how to complete each of the sections below.

## User Roles

- **Guest**: A visitor who has not created an account
- **User**: A registered user who logs meals and browses recipes
- **Admin**: Manages the recipe and ingredient database

## User Stories

1. As a guest, I want to browse recipes, so that I can explore the app before signing up.
2. As a guest, I want to filter recipes by cook time, so that I can find meals that fit my schedule.
3. As a guest, I want to filter recipes by estimated cost, so that I can find meals within my budget.
4. As a user, I want to create a new recipe, so that I can save my own meals to the app.
5. As a user, I want to edit an existing recipe, so that I can update ingredients or instructions.
6. As a user, I want to delete a recipe, so that I can remove meals I no longer want.
7. As a user, I want to log what I ate today, so that I can track my daily nutrition intake.
8. As a user, I want to see my daily nutrient totals compared to recommended values, so that I know what I am missing.
9. As a user, I want to view a recipe detail modal, so that I can see full nutrition info without leaving the page.
10. As a user, I want to view my profile and set a weekly food budget, so that the app can suggest affordable meals.
11. As a user, I want to browse the nutrient encyclopedia, so that I can learn what each nutrient does.
12. As an admin, I want to reset the database to its default state, so that I can demo the app cleanly.
